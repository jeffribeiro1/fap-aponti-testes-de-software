Regras aplicadas nesta atividade

Funcionalidades principais

  • Autenticação por usuário e senha.
  • Validação das credenciais cadastradas.
  • Mensagem genérica para dados inválidos, sem expor informações sensíveis.
  • Bloqueio temporário após 3 tentativas consecutivas incorretas.
  • Logout disponível na área autenticada.

Requisitos não funcionais

  • Processamento local da autenticação, projetado para responder em menos de 2 segundos.
  • Compatibilidade com navegadores modernos.
  • Registro das tentativas de login no armazenamento local para auditoria.

Regras de negócio

  • Usuário e senha são obrigatórios.
  • A senha deve possuir no mínimo 8 caracteres.
  • Após o bloqueio, uma nova tentativa só é permitida depois de 15 minutos.
  • Áreas restritas só aparecem para usuários autenticados.

Premissas e restrições

  • Não há integração com redes sociais.
  • Não há recuperação de senha nesta versão.
  • Existem usuários pré-cadastrados para testes.
  • Os dados usam uma identificação exclusiva do ambiente de testes.
